# Architecture Overview

MIBO follows a **microservices architecture** where each service owns its domain, data, and business logic. Services communicate asynchronously through NATS message broker and are fronted by a YARP-based API gateway.

<SectionHeader title="Design Principles" description="Core architectural decisions that guide the platform" />

<Callout type="tip" title="Why Microservices?">
  MIBO is built as a microservices platform so that each domain (identity, conversation, finance) can evolve, scale, and deploy independently without impacting the rest of the system.
</Callout>

<SectionHeader title="Service Ports" description="Default ports for local development" />

| Service | Default Port | Description |
|---------|-------------|-------------|
| API Gateway | 5000 | Entry point for all API calls |
| Identity Service | 5001 | Authentication & users |
| Conversation Service | 5002 | Chat & AI orchestration |
| Finance Service | 5003 | Financial data management |
| Client (SPA) | 5173 | React frontend |
| Docs | 3030 | Documentation site |

<Callout type="info" title="Port Configuration">
  All ports are configurable via `launchSettings.json` or environment variables. The table above shows the defaults used during local development.
</Callout>
