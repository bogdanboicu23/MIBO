# Services

## Backend applications

| Service | Stack | Responsibilities | Primary persistence |
| --- | --- | --- | --- |
| `MIBO.ApiGateway` | ASP.NET Core + Ocelot | Public HTTP entry point, JWT validation, CORS, claims-to-header propagation | none |
| `MIBO.IdentityService` | ASP.NET Core + ASP.NET Identity + EF Core | Login, registration, refresh flow, `me`, finance connection state, Spotify OAuth | PostgreSQL + Redis |
| `MIBO.ConversationService` | ASP.NET Core | Conversation CRUD, chat endpoints, SSE proxying, assistant/user message persistence, monitoring updates | MongoDB |
| `MIBO.ActionService` | ASP.NET Core | Tool/data access via `query` and `execute`, handler routing, retry integration, status summary | Redis + MongoDB for monitoring data |
| `MIBO.LangChainService` | FastAPI + LangGraph + Groq | Intent detection, execution planning, tool invocation, response composition, context compaction | MongoDB for conversation context windows |
| `MIBO.AppHost` | .NET Aspire | Local orchestration for services and infrastructure | none |

## Important service notes

### `MIBO.ApiGateway`

- Uses `ocelot.json` and environment-specific Ocelot config files.
- Injects `X-User-Id` and `X-User-Email` into downstream requests.

### `MIBO.IdentityService`

- Stores users in PostgreSQL through ASP.NET Identity.
- Issues access tokens in the response body and refresh tokens in an HTTP-only cookie.
- Stores Spotify tokens and finance connection flags in Redis.

### `MIBO.ConversationService`

- Exposes `GET/POST/PATCH/DELETE /api/v1/conversations*`.
- Exposes `POST /api/v1/chat` and a compatibility `POST /api/chat`.
- Persists user messages before AI execution and assistant payloads after the terminal `done` event.
- Owns conversation scoping, title generation, SSE relay, and synthetic monitoring for `platform` and `ai-agent`.

### `MIBO.ActionService`

- Exposes `POST /api/actions/query` and `POST /api/actions/execute`.
- Hosts handlers for products, weather, Spotify, finance, and Pomodoro.
- Can persist integration monitoring state and queue retries through RabbitMQ when configured.

### `MIBO.LangChainService`

- Exposes a single `/chat` SSE endpoint.
- Runs the sequential LangGraph pipeline:
  `memory_loader -> intent_agent -> planner_agent -> executor_agent -> composer_agent -> memory_saver`
- Reads memory directly from MongoDB, compacts older context when token pressure grows, and delegates typed data access to `ActionService`.
- Produces structured frontend-oriented JSON rather than a plain-text-only answer stream.
