# Chat & AI Overview

The current MIBO assistant is a coordinated runtime, not a single chatbot endpoint. It is composed of:

- a React client that opens SSE streams and materializes generative UI
- `MIBO.ApiGateway` for authentication and request routing
- `MIBO.ConversationService` for conversation ownership, persistence, SSE relay, and monitoring
- `MIBO.LangChainService` for AI reasoning, tool orchestration, and final response composition
- `MIBO.ActionService` for typed data access and action execution

## What the runtime is optimized for

The system is designed to return a structured assistant payload rather than only plain text. A final response can contain:

- `text`
- `components`
- `data_sources`
- `actions`
- `subscriptions`

That payload is then translated by the client into `ui.v1`, which allows the interface to keep working after the initial AI answer.

## End-to-end request path

```text
Browser
  -> ApiGateway
  -> ConversationService
  -> LangChainService
  -> ActionService
  -> external providers

Browser
  <- SSE relay
  <- ConversationService
  <- LangChainService
```

## Runtime responsibilities by service

| Layer | Runtime owner | What it does |
| --- | --- | --- |
| Conversation entry | `MIBO.ConversationService` | validates prompt, resolves conversation, persists user message, opens upstream AI stream |
| AI reasoning | `MIBO.LangChainService` | loads memory, classifies intent, plans execution, runs tools, composes final structured response |
| Tool and data execution | `MIBO.ActionService` | executes handlers for products, finance, weather, Spotify, and Pomodoro |
| Response delivery | `MIBO.ConversationService` | relays SSE downstream and persists the assistant payload on `done` |
| UI realization | `MIBO.Client/client` | parses the final payload and builds `ui.v1` from it |

## Core implementation characteristics

- The transport between browser and backend is **HTTP + SSE**.
- `ConversationService` is the outer orchestration boundary seen by the client.
- `LangChainService` is the inner AI boundary seen by `ConversationService`.
- The agent reads conversation context directly from MongoDB.
- The agent does not persist chat messages itself; `ConversationService` owns message persistence.
- The agent composes JSON that is intentionally shaped for the frontend sandbox registry.

## The two most important service boundaries

### `MIBO.ConversationService`

This service owns:

- conversation CRUD
- per-user ownership checks
- user and assistant message persistence
- SSE relay to the browser
- correlation and synthetic monitoring records for both platform and AI agent activity

### `MIBO.LangChainService`

This service owns:

- the `/chat` AI endpoint
- the LangGraph pipeline
- prompt contracts for intent, planning, composition, and context compaction
- tool invocation through `ActionService`
- response normalization when model output is imperfect JSON
