<SectionHeader title="Step-by-Step" description="Each phase of the conversation pipeline explained in detail" />

<Steps>
  <Step title="Message Receipt">
    The client sends a `ChatRequest` with the conversation ID, user ID, and prompt:

    ```json
    {
        "conversationId": "conv-123",
        "userId": "user-456",
        "prompt": "Show me my expenses this month"
    }
    ```
  </Step>
  <Step title="Planning Phase">
    The Chat Orchestrator delegates to the Planner, which uses an LLM to:
    - Analyze the user's intent
    - Select appropriate tools from the Tool Registry
    - Determine the execution order
  </Step>
  <Step title="Tool Execution">
    Selected tools are executed against backend services. Each tool is defined with:

    ```csharp
    ToolDefinition {
        name: "get_expenses",
        description: "Get user expenses with filters",
        method: "GET",
        urlTemplate: "/api/expenses/user/{userId}",
        requiredArgs: ["userId"]
    }
    ```
  </Step>
  <Step title="Response Composition">
    Two composers work in parallel:

    <FeatureGrid columns={2}>
      <FeatureCard title="Text Composer" description="Generates a natural language summary of the results" icon="вңҚпёҸ" />
      <FeatureCard title="UI Composer" description="Creates dynamic UI component specifications for rich rendering" icon="рҹҺЁ" />
    </FeatureGrid>
  </Step>
  <Step title="Delivery">
    The response is delivered back through the ChatController with both text and UI data:

    ```json
    {
        "text": "Here are your expenses for this month...",
        "uiV1": { /* generative UI specification */ },
        "correlationId": "corr-789"
    }
    ```
  </Step>
</Steps>

<SectionDivider />

<SectionHeader title="Action Handling" description="How user interactions with generative UI components are processed" />

When users interact with generative UI components (clicking buttons, applying filters), actions are sent via the ActionController:

```json
{
    "schema": "action.v1",
    "conversationId": "conv-123",
    "userId": "user-456",
    "action": {
        "type": "ui.setFilter",
        "payload": { "category": "groceries" }
    }
}
```

<Callout type="info" title="Event-Driven Processing">
  The Action Router processes these actions and publishes events via NATS JetStream, enabling decoupled handling across services.
</Callout>
