# Streaming

Both `LangChainService` and `ConversationService` use `text/event-stream`.

## Event types seen by the client

| Type | Emitted by | Meaning |
| --- | --- | --- |
| `session` | ConversationService or LangChainService | communicates the created conversation/session ID |
| `status` | LangChainService | intermediate progress message |
| `chunk` | LangChainService | raw partial fragment of the final JSON payload |
| `done` | LangChainService | complete final payload serialized as JSON text |
| `error` | optional | error during streaming or composition |

## Generic format

```text
data: {"type":"status","content":"Understanding your request..."}

data: {"type":"chunk","content":"{\"text\":\"..."}

data: {"type":"done","content":"{\"text\":\"Final response\",\"components\":[]}"}
```

## What ConversationService does with the stream

- reads the raw upstream agent stream
- detects the `done` event
- attempts to parse the final payload
- extracts the assistant text and payload
- persists the assistant message after the upstream stream completes
- emits a synthetic `session` event itself when it had to create the conversation before calling the agent

## What the React client does

`useChat.ts`:

1. reads the stream through `ReadableStream`
2. splits events on `\n\n`
3. updates the assistant status message for `status`
4. accumulates `chunk` fragments
5. parses the final `AgentFinalResponse` on `done`
6. builds `ui.v1` through `buildUiFromAgentResponse()`

## Final payload schema in practice

The `done` event usually contains a JSON string that can include:

- `text`
- `components`
- `data_sources`
- `actions`
- `subscriptions`

That makes the streaming contract an application protocol for generative UI, not just a text transport.

## Operational note

<Callout type="info" title="The assistant streams structured state">
  The main value of the stream is not only progressive text. It is the progressive delivery of a final structured assistant payload that can drive generative UI and follow-up actions.
</Callout>
