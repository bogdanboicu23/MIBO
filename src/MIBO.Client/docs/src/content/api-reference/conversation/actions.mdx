# Conversation Management

`ConversationService` exposes full conversation CRUD for the authenticated user.

## Endpoints

| Method | Route | Role |
| --- | --- | --- |
| `GET` | `/api/v1/conversations` | list the user's conversations |
| `POST` | `/api/v1/conversations` | create a new conversation |
| `GET` | `/api/v1/conversations/{conversationId}` | return conversation summary and messages |
| `PATCH` | `/api/v1/conversations/{conversationId}` | rename a conversation |
| `DELETE` | `/api/v1/conversations/{conversationId}` | delete the conversation, messages, and UI instances |

## List conversations

Query parameters:

- `skip`, default `0`
- `limit`, default `50`

Real implementation note:

- `MongoConversationStore` clamps `limit` to a maximum of `100`

## Create conversation

Request body:

```json
{
  "title": "optional"
}
```

If the title is missing, the service creates a conversation titled `New chat`.

## Get conversation details

Query parameters:

- `messagesLimit`, default `200`

Real implementation note:

- `MongoConversationStore` clamps `messagesLimit` to a maximum of `500`

Typical response:

```json
{
  "conversation": {
    "conversationId": "id",
    "userId": "user",
    "title": "Title",
    "createdAt": "2026-03-23T00:00:00Z",
```

## Important ownership rules

- ownership is enforced in `MongoConversationStore`
- if a conversation belongs to another user, the service returns `404`, not `403`
- delete removes the conversation document, message documents, and related UI instances

## Naming behavior

- default title is `New chat`
- titles are trimmed and normalized
- automatic title generation is based on the first user message if the conversation still has the default title
