# API Reference

All API requests go through the **Ocelot API Gateway**, which routes them to the appropriate microservice. The gateway runs on port 5000 in development and at `https://api.mibo.monster` in production.

<SectionDivider />

<SectionHeader title="Authentication" description="How to authenticate your API requests" />

Most endpoints require a JWT Bearer token. Include it in the `Authorization` header:

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

<Callout type="info" title="Getting a Token">
  Obtain tokens via the [Auth endpoints](/api-reference/identity/auth). Tokens are issued on login and can be refreshed using the refresh token cookie.
</Callout>

<SectionDivider />

<SectionHeader title="Gateway Routes" description="How the API Gateway maps upstream paths to downstream microservices" />

| Upstream Pattern | Downstream Service | Port (Dev) |
|-----------------|-------------------|------------|
| `/api/auth/*` | Identity Service | 5001 |
| `/api/users/*` | Identity Service | 5001 |
| `/api/v1/chat` | Conversation Service | 7251 |
| `/api/v1/*` | Conversation Service | 7251 |
| `/api/conversations/*` | Conversation Service | 7251 |
| `/hubs/ui` | Conversation Service (SignalR) | 7251 |
| `/api/accounts/*` | Finance Data Service | 5003 |
| `/api/transactions/*` | Finance Data Service | 5003 |
| `/api/expenses/*` | Finance Data Service | 5003 |
| `/api/budgets/*` | Finance Data Service | 5003 |
| `/api/summary/*` | Finance Data Service | 5003 |
| `/api/analytics/*` | Finance Data Service | 5003 |

<SectionHeader title="Common Response Patterns" description="Standard response shapes used across all endpoints" />

### Paginated Lists

```json
{
    "items": [...],
    "total": 142,
    "skip": 0,
    "limit": 30
}
```

### Error Responses

```json
{
    "message": "Error description",
    "details": "Additional context"
}
```

<Callout type="warning" title="Rate Limiting">
  All endpoints are subject to rate limiting through the API Gateway. Exceeding limits will return a `429 Too Many Requests` response.
</Callout>
