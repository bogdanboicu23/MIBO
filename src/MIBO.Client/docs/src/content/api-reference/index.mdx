# API Reference

MIBO has two contract layers that matter operationally:

1. **public HTTP endpoints** exposed through `MIBO.ApiGateway`
2. **internal action runtime contracts** exposed by `MIBO.ActionService` and consumed by both the AI agent and the interactive client

## Public surface exposed through the gateway

| Group | Upstream prefix | Auth |
| --- | --- | --- |
| Gateway routing | `/api/*` | varies by route |
| Identity | `/api/auth/*` | mixed |
| Main chat | `/api/v1/chat` | JWT |
| Conversations | `/api/v1/conversations*` | JWT |
| Action runtime | `/api/actions/*` | JWT for `query` and `execute` |
| Action status | `/api/actions/status/*` | currently public |

## Internal surface used by the AI runtime

The browser does **not** call the Python agent directly.

Instead:

- `ConversationService` calls `POST /chat` on `LangChainService`
- `LangChainService` emits `text/event-stream`
- `ConversationService` relays that stream to the browser
