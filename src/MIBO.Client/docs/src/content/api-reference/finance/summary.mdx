# Finance Summary

## Primary handler

| Handler | Downstream call | Arguments |
| --- | --- | --- |
| `finance.summary.get` | `GET /api/summary/user/{userId}` | `userId`, `user_id` |

## Runtime behavior

- `userId` is required
- if it is missing, Action Service throws `InvalidOperationException`
- the LangChain tool layer uses this handler through the finance tool wrappers

## Secondary synthetic handler

`finance.user.summary` exists only inside `ActionRouter` as a local deterministic data generator. It is not the primary external finance integration path.
