# Finance Accounts

These contracts are reached through `POST /api/actions/query`.

## Supported handlers

| Handler | Downstream call | Arguments |
| --- | --- | --- |
| `finance.accounts.list` | `GET /api/accounts` on BankService | `userId`, `user_id`, `limit`, `skip`, `sortBy`, `order` |
| `finance.accounts.get` | `GET /api/accounts/{id}` on BankService | `id`, `accountId` |

## Field hints returned by Action Service

- `entityType = account_list` or `account`
- `collectionPath = items` for list responses
- `labelField = accountName`
- `valueField = balance`
- `categoryField = type`

## Implementation notes

- `finance.accounts.get` throws if `id <= 0`
- authenticated calls can rely on `user_id` being injected by the gateway plus Action Service header propagation chain
