# Helm Charts

Each backend service has a Helm chart under `deploy/helm/`.

## Available charts

| Chart | Purpose |
| --- | --- |
| `action-service` | deploys the action runtime |
| `api-gateway` | deploys the Ocelot gateway |
| `app-host` | deployment scaffold for AppHost-related packaging |
| `conversation-service` | deploys the conversation runtime |
| `identity-service` | deploys the auth service |
| `langchain-service` | deploys the Python AI runtime |

## Typical chart structure

```text
deploy/helm/{service}/
├── Chart.yaml
├── values.yaml
└── templates/
    ├── deployment.yaml
    ├── service.yaml
    ├── hpa.yaml
    └── ingress.yaml   # only where applicable
```

## Environment overrides

The test environment overrides chart defaults with:

```text
deploy/environments/test/
├── values-action-service.yaml
├── values-api-gateway.yaml
├── values-app-host.yaml
├── values-conversation-service.yaml
├── values-identity-service.yaml
└── values-langchain-service.yaml
```

## What the overrides currently configure

- DigitalOcean registry images
- test/staging-style image tags
- connection strings and secret references
- environment variables such as:
  - `ACTION_SERVICE_URL`
  - `AGENT_SERVICE_URL`
  - `BANKSERVICE_BASE_URL`
  - Redis, JWT, Turnstile, Spotify, and Groq-related settings

## Deployment behavior in CI/CD

The backend workflows typically run:

1. shared test workflow
2. image build and push
3. `helm upgrade --install`
4. rollout wait and basic status dump
