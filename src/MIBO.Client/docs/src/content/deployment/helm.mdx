# Helm Charts

Each MIBO microservice has its own Helm chart under `deploy/helm/`, providing templated Kubernetes resources with configurable values.

<SectionDivider />

<SectionHeader title="Chart Overview" description="Available Helm charts and their default configurations." />

| Chart | Version | Default Port | Registry |
|-------|---------|-------------|----------|
| `api-gateway` | 0.1.0 | 8080 | `mibo/api-gateway` |
| `app-host` | 0.1.0 | 8080 | `mibo/app-host` |
| `conversation-service` | 0.1.0 | 8080 | `registry.digitalocean.com/mibo/conversation-service` |
| `identity-service` | 0.1.0 | 8080 | `registry.digitalocean.com/mibo/identity-service` |
| `langchain-service` | 0.1.0 | 8088 | `registry.digitalocean.com/mibo/langchain-service` |

<SectionDivider />

<SectionHeader title="Chart Structure" description="Each chart follows the standard Helm layout." />

```
deploy/helm/{service-name}/
├── Chart.yaml           # Chart metadata (name, version)
├── values.yaml          # Default configuration values
└── templates/
    ├── deployment.yaml  # Kubernetes Deployment
    ├── service.yaml     # Kubernetes Service (ClusterIP)
    ├── ingress.yaml     # Ingress resource (optional)
    └── hpa.yaml         # Horizontal Pod Autoscaler
```

### HPA Default Configuration

```yaml
autoscaling:
  enabled: true
  minReplicas: 1
  maxReplicas: 5
  targetCPUUtilizationPercentage: 70
```

### Ingress Default Configuration

```yaml
ingress:
  enabled: false
  className: traefik
  hosts:
    - host: service.test.example.com
      paths:
        - path: /
          pathType: Prefix
```

<SectionHeader title="Values Configuration" description="Per-service values files controlling deployment behavior." />

### API Gateway

```yaml
replicaCount: 1
image:
  repository: mibo/api-gateway
  tag: latest
  pullPolicy: IfNotPresent
service:
  port: 8080
resources:
  requests: { cpu: 100m, memory: 128Mi }
  limits:   { cpu: 500m, memory: 512Mi }
autoscaling:
  enabled: true
  minReplicas: 1
  maxReplicas: 5
  targetCPUUtilizationPercentage: 70
```

### Identity Service

The identity service has additional configuration for JWT and security:

```yaml
image:
  repository: registry.digitalocean.com/mibo/identity-service
  tag: test-fix-amd64
env:
  ASPNETCORE_ENVIRONMENT: Development
  ASPNETCORE_URLS: http://+:8080
  Jwt__Issuer: MIBO.IdentityService
  Jwt__Audience: MIBO.Services
  Jwt__AccessTokenExpirationMinutes: "15"
  Jwt__RefreshTokenExpirationDays: "7"
  Turnstile__Enabled: "true"
secrets:
  - name: ConnectionStrings__DefaultConnection
    secretName: database-secret
    secretKey: connection-string
  - name: Jwt__Key
    secretName: identity-secrets
    secretKey: jwt-key
```

<Callout type="danger" title="Sensitive Configuration">
  JWT keys and database connection strings are injected via Kubernetes Secrets. Never hardcode these values in the values files committed to version control.
</Callout>

### Langchain Service

The Langchain service uses custom health probe paths:

```yaml
image:
  repository: registry.digitalocean.com/mibo/langchain-service
  pullPolicy: Always
service:
  port: 8088
healthProbe:
  path: /langchain/healthz
  initialDelaySeconds: 5
  periodSeconds: 10
  timeoutSeconds: 5
  failureThreshold: 6
```

<SectionDivider />

<SectionHeader title="Environment Overrides" description="Per-environment value files that override chart defaults." />

The `deploy/environments/test/` directory contains per-service value files that override defaults:

```bash
deploy/environments/test/
├── kustomization.yaml
├── values-api-gateway.yaml          # tag: staging
├── values-app-host.yaml             # tag: staging
├── values-conversation-service.yaml # tag: staging
├── values-identity-service.yaml     # Full JWT + DB config
└── values-langchain-service.yaml    # tag: staging
```

<Callout type="tip" title="Deploying a Specific Image Tag">
  To deploy a service with a specific image tag, update the corresponding values file and apply with `helm upgrade` or through the Kustomize overlay.
</Callout>
