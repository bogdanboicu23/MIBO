# Deployment

MIBO runs on **Kubernetes** (DigitalOcean) with Helm charts, Traefik ingress, automatic TLS, and a full observability stack. The platform supports both local development with Docker Compose and production-grade Kubernetes deployments.

<SectionDivider />

<SectionHeader title="Production Architecture" description="End-to-end traffic flow from the internet through Traefik ingress to backend microservices and data stores." />

```
                         Internet
                            │
                    ┌───────▼────────┐
                    │  DigitalOcean  │
                    │  Load Balancer │
                    └───────┬────────┘
                            │  :80 / :443
                    ┌───────▼────────┐
                    │    Traefik     │  Ingress Controller
                    │   (v3.1)      │  Let's Encrypt TLS
                    └───────┬────────┘
                            │
           ┌────────────────┼────────────────┐
           │                │                │
   mibo.monster    api.mibo.monster   grafana.mibo.monster
           │                │                │
  ┌────────▼──────┐ ┌──────▼───────┐ ┌──────▼──────┐
  │  MIBO Client  │ │ API Gateway  │ │   Grafana   │
  │  (2 replicas) │ │ (2 replicas) │ │             │
  └───────────────┘ └──────┬───────┘ └─────────────┘
              ┌────────────┼────────────┐
              ▼            ▼            ▼
     ┌─────────────┐ ┌──────────┐ ┌──────────┐
     │  Identity   │ │  Convo   │ │ Langchain│
     │  Service    │ │ Service  │ │ Service  │
     └──────┬──────┘ └────┬─────┘ └──────────┘
            │             │
     ┌──────▼──────┐ ┌────▼─────┐ ┌──────────┐
     │ PostgreSQL  │ │  MongoDB │ │   NATS   │
     │             │ │          │ │JetStream │
     └─────────────┘ └──────────┘ └──────────┘
                                  ┌──────────┐
                                  │  Redis   │
                                  └──────────┘
```

|---------|-----|-------------|

<SectionHeader title="Container Registry" description="All service images are stored in the DigitalOcean Container Registry." />

```
registry.digitalocean.com/mibo/
├── client:test-latest
├── api-gateway:latest
├── identity-service:test-latest
├── conversation-service:test-latest
├── langchain-service:test-latest
└── ...
```

Images are pulled using the `do-registry-secret` Kubernetes secret.

<Callout type="warning" title="Never Commit Secrets">
  Never commit secrets to the repository. All sensitive values are injected via Kubernetes Secrets and referenced by name in deployment manifests.
</Callout>

<SectionDivider />

<SectionHeader title="CI/CD" description="Automated deployment pipeline with scoped Kubernetes permissions." />

A dedicated `ci-deployer` service account with scoped RBAC permissions handles automated deployments. It has access to manage deployments, services, ingresses, and Traefik resources in both the `mibo` and `traefik` namespaces.
