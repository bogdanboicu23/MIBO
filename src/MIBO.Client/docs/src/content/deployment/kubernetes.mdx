# Kubernetes Layout

The Kubernetes material in the repository is split across two patterns:

1. Helm charts and environment values under `deploy/`
2. raw manifests for frontend apps and Traefik under `k8s/`

## Active test-environment layout

The main environment represented by the current workflows is the `test` namespace.

## Repository structure

```text
deploy/
├── helm/                       # backend service charts
├── environments/test/          # test values used by CI/CD
└── migrations/                 # DB migration jobs

k8s/
├── apps/                       # client/docs/status manifests
├── traefik/                    # ingress controller and dashboard
└── ci/                         # CI service account and RBAC
```

## What is applied where

### Backend services

Applied via Helm from GitHub Actions using:

- `deploy/helm/*`
- `deploy/environments/test/values-*.yaml`

### Frontend/docs/status

Applied via direct `kubectl apply` using:

- `k8s/apps/10-mibo-client.yaml`
- `k8s/apps/12-mibo-docs.yaml`
- `k8s/apps/14-mibo-status.yaml`
- their related ingress manifests

### Cluster edge and ingress

Managed through:

- `k8s/traefik/*`
- `k8s/ci/*`

## Operational source of truth

For deployment behavior, use the GitHub Actions workflows together with:

- `deploy/helm/*`
- `deploy/environments/test/*`
- `k8s/apps/*` for frontend applications
- `k8s/traefik/*` for ingress resources
