# Ingress & TLS

MIBO uses Traefik as the ingress controller for the environment represented by this repository.

## Public hosts represented by current manifests

| Host | Backing service | Manifest source |
| --- | --- | --- |
| `mibo.monster` | `mibo-client` | `k8s/apps/11-mibo-client-ingress.yaml` |
| `docs.mibo.monster` | `mibo-docs` | `k8s/apps/13-mibo-docs-ingress.yaml` |
| `status.mibo.monster` | `mibo-status` | `k8s/apps/15-mibo-status-ingress.yaml` |
| `api.mibo.monster` | `api-gateway` | Helm values for `api-gateway` |
| `traefik.mibo.monster` | Traefik dashboard | `k8s/traefik/05-06*` |

## TLS behavior

- Traefik is configured with Let's Encrypt certificate resolution
- HTTP routes are redirected to HTTPS
- Traefik dashboard access is protected with basic auth middleware

## Practical note

The frontend apps use raw Kubernetes ingress manifests, while the API Gateway ingress is expressed through Helm chart values. That split is intentional in the current repository layout.
