# Observability

The current operational picture of MIBO is centered on status monitoring, service logs, and deployment rollouts.

## Active observability features

### Integration status dashboard

The clearest operational status surface today is the status flow:

- `ActionService` exposes `GET /api/actions/status/summary`
- monitoring data is stored in MongoDB
- `MIBO.Client/status` renders the results

### Synthetic platform monitoring

`ConversationService` writes synthetic monitoring records for:

- `platform`
- `ai-agent`

This allows the status page to cover both third-party integrations and the conversation and AI path itself.

## Practical operational checks

For day-to-day validation, use:

1. the status dashboard and ActionService monitoring data
2. service logs from the running environment
3. rollout status from GitHub Actions or `kubectl`
