# Installation

There are two practical ways to run MIBO locally:

1. the recommended **AppHost** path, which starts the full platform topology
2. a **manual service-by-service** path, useful when you are working on a single service or frontend

## Prerequisites

| Tool | Why it is needed |
| --- | --- |
| .NET 9 SDK | Required by the .NET services, AppHost, and tests |
| Node.js 20+ and npm | Required by `MIBO.Client/client`, `MIBO.Client/docs`, and `MIBO.Client/status` |
| Docker | Required by AppHost for infrastructure and for the Python agent container |
| Python 3.11+ | Only needed if you want to run `MIBO.LangChainService` outside Docker |

## Recommended startup: AppHost

The local source of truth for orchestration is `src/MIBO.AppHost/AppHost.cs`.

### Required environment

AppHost requires a Groq API key to start the LangChain service:

```bash
export GROQ_API_KEY=your_key_here
```

### Start the platform

```bash
cd src/MIBO.AppHost
dotnet run
```

### What AppHost provisions

- Redis
- MongoDB
- PostgreSQL
- RabbitMQ
- `action-service`
- `langchain-service`
- `conversation`
- `identity-service`
- `api-gateway`
- `client`
- `docs`
- `status`

### Why this path is preferred

- It reflects the current local dependency graph.
- It injects the service URLs and connection strings expected by the running services.
- It avoids manual setup for MongoDB, Redis, PostgreSQL, RabbitMQ, and the Python agent.

## Manual startup

Use this only if you need to work on isolated parts of the platform.

### Backend services

```bash
cd src
dotnet run --project MIBO.IdentityService
dotnet run --project MIBO.ActionService
dotnet run --project MIBO.ConversationService
dotnet run --project MIBO.ApiGateway
```

### LangChain service

```bash
cd src/MIBO.LangChainService
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m uvicorn main:app --host 0.0.0.0 --port 8088 --proxy-headers
```

You must provide at least:

- `GROQ_API_KEY`
- `ACTION_SERVICE_URL`
- Mongo-related environment variables if you want agent memory compaction to work

### Frontends

```bash
cd src/MIBO.Client/client
npm install
npm run dev
```

```bash
cd src/MIBO.Client/docs
npm install
npm run dev
```

```bash
cd src/MIBO.Client/status
npm install
npm run dev
```

## Useful validation commands

```bash
dotnet test tests/MIBO.Tests.sln
```

```bash
cd src/MIBO.Client/docs
npm run build
```
