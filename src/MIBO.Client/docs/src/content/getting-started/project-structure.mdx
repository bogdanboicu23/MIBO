# Project Structure

The repository is organized around a single `src/` workspace with backend services, frontend applications, shared libraries, and local orchestration. Deployment and CI/CD assets live at the repository root.

## Top-level repository layout

```text
MIBO/
├── src/                  # Application code, services, frontends, shared libraries
├── tests/                # Unit and end-to-end test projects
├── deploy/               # Helm charts, environment values, and migration jobs
├── k8s/                  # Kubernetes manifests for apps, ingress, and platform resources
├── .github/workflows/    # CI/CD pipelines
└── scripts/              # Local helper scripts
```

## `src/` layout

```text
src/
├── MIBO.sln
├── MIBO.AppHost/             # Aspire-based local orchestrator
├── MIBO.ApiGateway/          # Ocelot gateway
├── MIBO.IdentityService/     # Auth, JWT, Spotify OAuth, finance connection state
├── MIBO.ConversationService/ # Chat endpoints, SSE relay, conversation persistence
├── MIBO.ActionService/       # Tool/data runtime for query and execute contracts
├── MIBO.LangChainService/    # FastAPI + LangGraph AI pipeline
├── MIBO.Cache.Redis/         # Redis-backed finance and Spotify stores
├── MIBO.Storage.Mongo/       # Mongo-backed conversation and monitoring stores
├── MIBO.Client/
│   ├── client/               # Main React application
│   ├── docs/                 # Documentation site
│   └── status/               # External-service status dashboard
└── compose.yaml              # Container-based local runtime assets
```

## Service ownership by folder

| Path | Role |
| --- | --- |
| `src/MIBO.ApiGateway` | Public HTTP entry point and JWT validation |
| `src/MIBO.IdentityService` | Authentication, registration, refresh flow, `me`, finance connection markers, Spotify OAuth |
| `src/MIBO.ConversationService` | Conversation CRUD, chat endpoints, SSE proxying, persistence, platform and agent monitoring snapshots |
| `src/MIBO.ActionService` | Public `query` and `execute` runtime used by the agent and generative UI |
| `src/MIBO.LangChainService` | Intent, plan, execution, composition, and memory compaction pipeline |
| `src/MIBO.Cache.Redis` | Redis token and connection storage |
| `src/MIBO.Storage.Mongo` | Mongo bootstrap, indexes, conversation store, UI instance store, monitoring store |

## Frontend and docs ownership

| Path | Role |
| --- | --- |
| `src/MIBO.Client/client` | Authenticated end-user application |
| `src/MIBO.Client/docs` | Static docs application generated from MDX |
| `src/MIBO.Client/status` | Public status page for integration health |

## Infrastructure and delivery

| Path | Role |
| --- | --- |
| `deploy/helm/*` | Helm charts for backend services |
| `deploy/environments/test/*` | Test-environment values used by GitHub Actions deployments |
| `deploy/migrations/*` | Identity database migration jobs |
| `k8s/apps/*` | Raw manifests for client, docs, status, and supporting resources |
| `k8s/traefik/*` | Traefik namespace, deployment, dashboard, auth, and TLS storage |
| `.github/workflows/*` | Build, test, image publish, and deployment automation |

## Tests

The test solution is separate from the application solution:

```text
tests/
├── MIBO.Tests.sln
├── MIBO.E2ETests/               # End-to-end coverage for auth, conversations, streaming, and actions
├── MIBO.Storage.Mongo.Tests/    # Mongo storage and index coverage
├── MIBO.ActionService.Tests/    # Action service tests
├── MIBO.ApiGateway.Tests/       # Gateway tests
├── MIBO.ConversationService.Tests/
├── MIBO.IdentityService.Tests/
├── MIBO.LangChainService.Tests/
└── TESTING-GUIDE.md             # Repository testing guide
```
