# Project Structure

The MIBO repository is organized as a monorepo with all microservices and the frontend client in a single repository.

<SectionDivider />

<SectionHeader title="Root Layout" description="Top-level directory structure of the MIBO monorepo" />

```
MIBO/
├── src/
│   ├── MIBO.Gateway/              # YARP API Gateway
│   ├── MIBO.IdentityService/      # Authentication & user management
│   ├── MIBO.ConversationService/  # Chat orchestration & AI
│   ├── MIBO.FinanceService/       # Financial data management
│   ├── MIBO.Shared/               # Shared libraries & contracts
│   └── MIBO.Client/
│       ├── client/                # React SPA (main app)
│       └── docs/                  # Documentation site (you are here)
├── docker-compose.yml
└── MIBO.sln
```

<SectionHeader title="Service Internal Structure" description="Every .NET microservice follows this consistent folder layout" />

```
MIBO.{ServiceName}/
├── Controllers/       # API endpoints
├── Services/          # Business logic
├── Models/            # Domain models & DTOs
├── Data/              # EF Core DbContext & migrations
├── Messaging/         # NATS publishers/subscribers
├── Program.cs         # Service bootstrap
└── appsettings.json   # Configuration
```

<Callout type="info" title="Consistent Conventions">
  All services follow the same folder structure. This makes it easy to navigate between services and understand where to find specific functionality regardless of which service you are working in.
</Callout>

<SectionDivider />

<SectionHeader title="Client Application Structure" description="The React SPA that powers the MIBO user interface" />

```
client/
├── src/
│   ├── components/
│   │   ├── ui/        # Reusable UI components (Button, Card, Input, etc.)
│   │   ├── chat/      # Chat-specific components
│   │   └── finance/   # Finance-specific components
│   ├── hooks/         # Custom React hooks
│   ├── utils/         # Utility functions
│   ├── theme/         # Theme tokens & configuration
│   ├── services/      # API client services
│   └── pages/         # Route pages
├── public/
└── package.json
```
