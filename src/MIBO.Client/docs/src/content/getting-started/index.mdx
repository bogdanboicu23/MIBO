# MIBO

MIBO is a multi-application, multi-service platform centered around a conversational client, a .NET orchestration layer, a structured action runtime, and a Python LangGraph agent. The repository contains the live platform code, deployment assets, CI/CD workflows, shared libraries, tests, and the documentation site itself.

## What exists today

| Area | Current implementation |
| --- | --- |
| Public entry point | `MIBO.ApiGateway` using **Ocelot** |
| Authentication | `MIBO.IdentityService` with ASP.NET Identity, PostgreSQL, JWT access tokens, and refresh-token cookies |
| Conversation runtime | `MIBO.ConversationService` with MongoDB-backed conversation and message persistence |
| AI runtime | `MIBO.LangChainService` built with FastAPI, LangGraph, Groq, and structured JSON output |
| Tool/data runtime | `MIBO.ActionService` exposing `query` and `execute` contracts for agent tools and generative UI |
| Frontends | React client, docs site, and an external-service status dashboard |
| Local orchestration | `.NET Aspire` AppHost provisioning services and infrastructure for local development |
| Deployment model | Helm for backend services, raw Kubernetes manifests for client/docs/status, Traefik ingress, GitHub Actions pipelines |

## Core capabilities

- Authenticated conversational interface with persistent conversation history.
- Structured SSE streaming from the AI pipeline to the client.
- Generative UI rendering based on `ui.v1` runtime payloads.
- Action-backed data integrations for products, weather, finance, Spotify, and Pomodoro.
- Monitoring snapshots and audit history for external integrations.
- Separate documentation and status applications deployed alongside the main client.
